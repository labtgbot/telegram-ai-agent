## Кратко

Composio-конфигурация хранится как произвольный dict без схемы/ограничений и
возвращается через GET `/composio` роли `analyst`. Если в конфиг попадут
секреты (ключи/токены), они будут раскрыты наименее привилегированной роли.

## Доказательства

- `backend/app/api/v1/admin_system.py:397-424` — запись произвольного dict без
  валидации схемы.
- `backend/app/api/v1/admin_system.py:230-240` — GET `/composio` доступен
  `analyst` и возвращает сохранённый конфиг как есть.

## Влияние

Латентное раскрытие секретов: отсутствие схемы допускает хранение
чувствительных полей, которые затем читает analyst. Defence-in-depth отсутствует.

## Предлагаемое исправление

- Валидировать конфиг по явной схеме (whitelist полей), маскировать/исключать
  секреты в ответе, поднять минимальную роль для чтения до `super_admin`.

## Критерии приёмки

- [ ] Только разрешённые поля принимаются и отдаются.
- [ ] Секреты не попадают в ответ; чтение требует повышенной роли.
- [ ] Тесты на схему и доступ.
