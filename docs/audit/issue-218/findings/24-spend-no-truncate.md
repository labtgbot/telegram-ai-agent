# TokenService.spend() не усекает composio_tool/mcp_server до 255 (robustness-долг)

Родительский контекст: #218

| Поле | Значение |

| Критичность | Low |

| Labels | `bug`, `tokens`, `backend`, `stage-3-low`, `complexity-low` |
| Status | Confirmed |
| GitHub issue | _pending_ |

## Кратко

`TokenService.spend()` пишет `composio_tool`/`mcp_server` без усечения до лимита
столбца (255), в отличие от других путей. Сейчас значения короче лимита, поэтому
дефект недостижим, но это robustness-долг: будущий провайдер с длинным
идентификатором вызовет ошибку вставки.

## Доказательства

- `backend/app/services/token_service.py:397-398` — присвоение без усечения до
  `String(255)`.

## Влияние

Латентный: при появлении длинных значений — `DataError`/обрыв транзакции
списания. Сейчас не воспроизводится.

## Предлагаемое исправление

- Усекать значения до длины столбца (как в смежных путях) либо валидировать
  длину на входе.

## Критерии приёмки

- [ ] Длинные `composio_tool`/`mcp_server` не ломают `spend()`.
- [ ] Тест с превышающим лимит значением.
