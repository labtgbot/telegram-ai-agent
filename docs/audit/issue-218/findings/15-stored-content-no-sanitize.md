# Сохранённый admin-контент не HTML-санитизируется → латентный stored XSS

## Кратко

Admin-контент (prompt templates, FAQ, welcome messages) сохраняется без
HTML-санитизации. Сейчас render-sink, интерпретирующего HTML, нет, поэтому это
латентный stored XSS: как только появится компонент, рендерящий этот контент как
HTML, появится и уязвимость.

## Доказательства

- `backend/app/api/v1/admin_content.py:129` — контент сохраняется без
  санитизации/экранирования.

## Влияние

Латентная: при добавлении HTML-render-sink сохранённый payload выполнится в
браузере администратора/пользователя. Defence-in-depth отсутствует.

## Предлагаемое исправление

- Санитизировать/экранировать пользовательский HTML на входе или строго
  экранировать на выводе; зафиксировать политику «контент — это текст, не HTML».

## Критерии приёмки

- [ ] Сохранение/рендер admin-контента не допускает исполнения внедрённого HTML.
- [ ] Тест с payload подтверждает экранирование.
