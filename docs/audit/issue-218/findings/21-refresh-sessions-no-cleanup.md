## Кратко

Серверная модель admin refresh sessions (введена в #211) не имеет процедуры
удаления истёкших/отозванных записей. Таблица растёт неограниченно с каждым
логином/ротацией.

## Доказательства

- `backend/app/models/admin_refresh_session.py` — модель без TTL/cleanup.
- Нет воркера/джобы, удаляющей истёкшие `admin_refresh_sessions`.

## Влияние

Неограниченный рост таблицы: деградация запросов аутентификации и рост объёма БД
со временем. Эксплуатации нет, но это операционный/гигиенический долг.

## Предлагаемое исправление

- Периодический cleanup истёкших/отозванных сессий (worker или SQL-джоба),
  индекс по `expires_at`, опционально партиционирование/ретенция.

## Критерии приёмки

- [ ] Истёкшие/отозванные сессии регулярно удаляются.
- [ ] Тест/джоба покрывает очистку.
