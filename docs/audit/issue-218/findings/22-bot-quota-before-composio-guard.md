# Bot: квота генерации списывается до guard на composio is None → сжигает квоту на выключенной фиче

Родительский контекст: #218

| Критичность | Low |
| Stage | Stage 3 - Low priority |

## Кратко

В bot-хендлерах генерации квота потребляется до проверки на `composio is None`.
Если Composio отключён/недоступен, квота уже списана, хотя запрос гарантированно
не будет выполнен.

## Доказательства

- `backend/app/bot/handlers.py:403/406` — порядок: потребление квоты прежде
  guard'а на `composio`.
- Аналогично `backend/app/bot/handlers.py:567/570` и `738/741`.

## Влияние

Пользователь теряет единицы квоты на фиче, которая отключена — несправедливое
расходование лимита без полезного результата.

## Предлагаемое исправление

- Перенести guard `composio is None` до потребления квоты; при отключённой фиче
  возвращать ошибку без списания.

## Критерии приёмки

- [ ] При `composio is None` квота не списывается.
- [ ] Тест проверяет отсутствие списания на отключённой фиче.
