# Admin login: code потребляется до TOTP-гейта → одна ошибка TOTP требует нового кода

## Кратко

В admin-login одноразовый code помечается потреблённым до проверки TOTP. Если
super_admin ошибётся в TOTP, code уже израсходован, и приходится запрашивать
новый — лишний out-of-band раунд и потенциальный grief/DoS на учётку
super_admin.

## Доказательства

- `backend/app/api/v1/admin_login.py:138-142` — code потребляется/инвалидируется.
- `backend/app/api/v1/auth.py:343-350,368-384` — TOTP проверяется уже после
  потребления кода.

## Влияние

Хрупкий UX и вектор grief: каждая ошибка TOTP сжигает code, вынуждая повторную
выдачу. Для super_admin это можно эксплуатировать как лёгкий annoyance/DoS.

## Предлагаемое исправление

- Потреблять code только после успешной проверки TOTP (атомарно завершать оба
  фактора), либо давать ограниченное число TOTP-попыток в пределах одного кода.

## Критерии приёмки

- [ ] Ошибка TOTP не инвалидирует валидный code в пределах допустимых попыток.
- [ ] Тест проверяет повтор TOTP без перевыпуска кода.
