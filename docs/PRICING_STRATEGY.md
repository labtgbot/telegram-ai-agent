# Pricing Strategy

## Принципы

1. **На 50% дешевле Mira** в среднем по аналогичным пакетам.
2. **Гибкость**: цены меняются администратором без релиза.
3. **Прозрачность**: цены и токены отображаются в одной шкале (Stars / Tokens).
4. **Виральность**: реферальная программа и daily bonus.

## Базовые пакеты

| Пакет | Tokens | Stars | Mira | Отрыв |
|-------|--------|-------|------|-------|
| Starter | 500 | 250 ⭐ | 500 ⭐ | -50% |
| Basic | 1,200 | 500 ⭐ | 1,000 ⭐ | -50% |
| Premium | 2,000 | 750 ⭐ | 1,500 ⭐ | -50% |
| Pro (мес.) | 2,000/мес | 500 ⭐ | 999 ⭐ | -50% |

## Динамические факторы

```python
PRICING_CONFIG = {
    "global_discount": 50,
    "seasonal_promo": 0,
    "first_purchase_bonus": 20,
    "referral_bonus": 100,
    "daily_bonus": 10,
}
```

Любое поле редактируется в Admin CRM → Pricing.

## Конверсия Stars → USD

Ориентир: 1 ⭐ ≈ $0.013 (обновляется через CRM по факту).

## A/B Эксперименты

В будущем — split-тесты на разные пакеты для разных когорт (через `admin_settings.experiments`).

## Edge Cases

- Действующий месячный план не пересчитывается при изменении цен (фиксированная стоимость в `subscriptions.last_transaction_id`).
- Активные предложения (`seasonal_promo > 0`) применяются ТОЛЬКО к будущим покупкам.
